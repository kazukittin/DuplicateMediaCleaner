# Windows専用 類似・重複画像/動画削除アプリケーション
## 要件定義書

---

## 1. プロジェクト概要

### 1.1 プロジェクト名
**DuplicateMediaCleaner** （仮称）

### 1.2 目的
Windowsユーザーが、ローカルストレージに保存された画像および動画ファイルから、重複ファイルおよび類似ファイルを検出し、安全に削除できるデスクトップアプリケーションを提供する。

### 1.3 対象ユーザー
- PCの写真や動画を整理したい個人ユーザー
- ストレージを効率化したい業務ユーザー
- 特別な技術知識を必要としない一般ユーザー

### 1.4 開発プラットフォーム
- **対象OS**: Windows 10以上
- **推奨スペック**: 
  - CPU: Intel Core i5相当以上
  - メモリ: 4GB以上（大規模スキャン時は8GB推奨）
  - ディスク空き容量: 1GB以上

## 2. 機能要件

### 2.1 ファイルスキャン機能

#### 2.1.1 スキャン対象範囲
- **フォルダ指定方式**: ユーザーが対象フォルダをダイアログで選択
- **サブフォルダ自動探索**: 指定フォルダ配下のすべてのサブフォルダを自動的に検索
- **複数フォルダ選択**: 将来的拡張性を考慮（初期版は単一フォルダ選択）

#### 2.1.2 対象ファイル形式

**画像ファイル**
- 拡張子: .jpg, .jpeg, .png, .gif, .bmp, .webp, .tiff, .ico
- EXIF情報の処理: メタデータを検出に活用

**動画ファイル**
- 拡張子: .mp4, .avi, .mov, .mkv, .flv, .wmv, .m4v, .webm
- フレームレート: 30fps を標準とする

#### 2.1.3 ファイル処理の優先度
- **画像と動画は独立して検出処理を実施**
- タブまたはセクション分けで結果を表示
- ユーザーが各タイプごとに削除判断を行える

### 2.2 重複ファイル検出機能

#### 2.2.1 完全重複の定義
- **ファイルハッシュ値が完全一致** (MD5 or SHA-256)
- ファイルサイズが同じ
- 類似度: 100%

#### 2.2.2 検出アルゴリズム
1. ファイルハッシュの計算
2. ハッシュ値が一致するファイルをグループ化
3. グループ内で最新ファイル（更新日時）を「保持ファイル」として標識

### 2.3 類似ファイル検出機能

#### 2.3.1 類似度の計算方法
- **画像**: 知覚ハッシュ（pHash）またはORB特徴量マッチング
  - 複数の検出手法を組み合わせて精度を向上
- **動画**: 先頭フレーム＋中盤フレーム＋終盤フレームを抽出し、画像として比較

#### 2.3.2 類似度スレッショルド
- **ユーザー調整可能**: 0～100%の範囲でスライダーバーで調整
- **デフォルト値**: 85%
- **推奨値の目安**:
  - 低感度 (60～70%): 大きく異なる画像も類似と判定（誤検出が増加）
  - 中感度 (80～90%): バランス型、一般的な使用推奨
  - 高感度 (95～100%): ほぼ同じ画像のみ検出（見落としが増加）

#### 2.3.3 類似ファイルのグループ化
- 類似度スレッショルドを超えるファイル同士をグループ化
- 各グループで「代表画像」（ファイルサイズが大きい、または更新日時が新しい）を設定
- その他のファイルは削除候補として標識

### 2.4 削除確認機能

#### 2.4.1 確認方法
- **カテゴリごと（類似度レベル別）に確認**
  - 100% (完全重複)
  - 95～99% (極度に類似)
  - 85～94% (よく似ている)
  - 70～84% (やや似ている)
  - 別途、カスタム範囲指定可能

#### 2.4.2 確認画面の構成
- **左ペイン**: カテゴリツリー（類似度レベル別フォルダ構造）
- **中央ペイン**: 該当ファイルの一覧表示
  - チェックボックスで削除対象を選択
  - ファイル情報 (名前、サイズ、更新日時、保存場所)
- **右ペイン**: プレビュー表示
  - 画像: サムネイル表示（最大200×200px）
  - 動画: 先頭フレーム表示（最大200×112px）
  - 類似度スコア表示

#### 2.4.3 複数選択機能
- 「このカテゴリすべて削除」ボタン
- 「選択アイテムを削除」ボタン
- Ctrl+クリックで複数選択対応

### 2.5 削除実行機能

#### 2.5.1 削除方式
- **通常削除**: Windowsごみ箱へ移動（復旧可能）
- **完全削除**: 上書き削除（復旧困難、オプション）

#### 2.5.2 削除前の安全対策
- 削除実行前に最終確認ダイアログ表示
- 削除対象ファイル数を表示
- 解放されるディスク容量を計算・表示

#### 2.5.3 削除実行中の表示
- プログレスバー表示
- 削除処理のキャンセル機能（実行中のみ有効）
- 処理ログのリアルタイム表示

#### 2.5.4 削除後の処理
- 削除成功/失敗の結果報告
- 実行ログをファイルとして保存（削除日時、削除ファイル、理由など）

## 3. 非機能要件

### 3.1 パフォーマンス
- **スキャン速度**: 10,000ファイル（合計10GB）を60分以内に処理
- **メモリ効率**: 常駐メモリ500MB以下、ピーク時1GB以下
- **UI応答性**: スキャン中も操作可能（別スレッド処理）

### 3.2 安全性
- **削除の取り消し**: ごみ箱から復旧できる設計
- **ファイルロック対策**: 使用中のファイルは検出するが削除しない
- **バックアップ推奨**: ユーザーに初回実行時にバックアップを促す警告

### 3.3 ユーザビリティ
- **日本語対応**: 完全な日本語UI
- **Undo機能**: 過去3回の削除処理を復元可能（オプション）
- **ホットキー**: Ctrl+Q（終了）など標準的なショートカット

### 3.4 ロギング・監視
- **操作ログ**: スキャン結果、削除実行内容をCSVで出力
- **エラーログ**: クラッシュ時のスタックトレース記録
- **ログ保存先**: %APPDATA%\DuplicateMediaCleaner\logs\

## 4. 技術仕様

### 4.1 推奨開発スタック

#### フロントエンド
- **フレームワーク**: Electron（クロスプラットフォーム対応、Web技術を活用）
  - または: WPF（Windows専用、ネイティブ性能）
- **UI フレームワーク**: React + TypeScript
- **スタイリング**: Tailwind CSS または Material-UI
- **アイコン**: Font Awesome または Material Icons

#### バックエンド（プロセス処理）
- **言語**: Python 3.9+（画像処理ライブラリが充実）
  - または: C# (.NET 6+)
- **画像処理**: OpenCV, PIL/Pillow
- **動画処理**: ffmpeg-python, MediaInfo
- **ハッシュ計算**: hashlib, cryptography
- **スレッド管理**: threading, asyncio

#### 外部ライブラリ
- **pHash**: imagehash
- **動画フレーム抽出**: ffmpeg-python
- **ハッシュ比較**: Levenshtein距離計算用 difflib
- **UI連携**: WebSocket (Electron ↔ Python)

### 4.2 アーキテクチャ

```
[Electron UI (React)] 
    ↓ (IPC/WebSocket)
[Python Backend Server]
    ↓
[ファイルシステム] [画像処理エンジン] [動画処理エンジン]
```

**理由**:
- Electronで拡張性を確保（将来的にMacOS対応も可能）
- Pythonの画像処理ライブラリをフルに活用
- UI と 重い処理を分離してレスポンス性を向上

### 4.3 データベース
- ローカルSQLite（スキャン結果キャッシュ）
- スキーマ: files, duplicates, similarities, scan_sessions

## 10. 用語定義

| 用語 | 定義 |
|------|------|
| **完全重複** | ファイルハッシュ値が100%一致するファイル |
| **類似ファイル** | 知覚ハッシュで計算された類似度がスレッショルド以上のファイル |
| **知覚ハッシュ (pHash)** | 画像内容を数値化し、視覚的な類似度を算出する手法 |
| **スレッショルド** | 類似と判定する基準値（0～100%） |
| **グループ** | 重複または類似の関係にあるファイルの集合 |
| **代表ファイル** | グループ内で保持する「基準」ファイル |

## 付録: 参考資料

### 画像処理アルゴリズム
- **pHash (Perceptual Hash)**
  - 画像を縮小 → グレースケール化 → DCT変換 → ハッシュ生成
  - 高速で精度が良い

- **ORB (Oriented FAST and Rotated BRIEF)**
  - 特徴点抽出と記述子生成
  - 回転に強く、pHashより精度が高い場合がある

### 動画処理
- **FFmpeg**: フレーム抽出用
- **フレーム間隔**: 先頭/中盤/終盤を標本化
