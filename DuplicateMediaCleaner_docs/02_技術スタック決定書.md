# DuplicateMediaCleaner 技術スタック決定書

## 1. 概要

本ドキュメントは、Windows専用の類似・重複画像/動画削除アプリケーション「DuplicateMediaCleaner」の技術スタックを定義する。

**基本方針:**
- フロントエンド: Electron + React + TypeScript
- バックエンド: Python 3.9+
- アーキテクチャ: Electron ↔ Python（IPC/WebSocket通信）

### 4.2 画像処理（推奨ライブラリ）

#### **4.2.1 画像読み込み・基本処理**
```python
# Pillow (PIL Fork)
from PIL import Image

# OpenCV
import cv2
```

| ライブラリ | 用途 | 特徴 |
|-----------|------|------|
| **Pillow** | 画像ファイル読み込み、メタデータ取得 | ✅ 軽量、多形式対応<br>✅ EXIF情報の読み取り<br>✅ サムネイル生成 |
| **OpenCV (opencv-python)** | 画像の特徴量抽出、比較 | ✅ 高速C++実装<br>✅ ORB, SIFT特徴量<br>✅ ヒストグラム比較 |
| **NumPy** | 画像配列操作 | ✅ OpenCV/Pillowとの連携 |

#### **4.2.2 知覚ハッシュ（類似画像検出の核）**
```python
import imagehash
from PIL import Image

# 知覚ハッシュ計算
phash = imagehash.phash(Image.open('image.jpg'))
dhash = imagehash.dhash(Image.open('image.jpg'))
whash = imagehash.whash(Image.open('image.jpg'))
```

| ライブラリ | 用途 | 特徴 |
|-----------|------|------|
| **imagehash** | 知覚ハッシュ計算 | ✅ pHash, dHash, wHash, aHash対応<br>✅ シンプルなAPI<br>✅ 類似度比較が容易 |

**使用するハッシュアルゴリズム:**
- **pHash (Perceptual Hash)**: メイン使用。DCT変換ベース、縮小・回転・色調変化に強い
- **dHash (Difference Hash)**: 補助。高速、微細な違いを検出
- **wHash (Wavelet Hash)**: 精度向上用。ウェーブレット変換ベース

#### **4.2.3 完全重複検出**
```python
import hashlib

# SHA-256ハッシュで完全重複検出
def get_file_hash(filepath):
    with open(filepath, 'rb') as f:
        return hashlib.sha256(f.read()).hexdigest()
```

| ライブラリ | 用途 |
|-----------|------|
| **hashlib (標準)** | SHA-256/MD5ハッシュ計算 |
| **xxhash** | 高速ハッシュ（大容量ファイル用） |

### 4.3 動画処理（推奨ライブラリ）

#### **4.3.1 動画読み込み・フレーム抽出**
```python
# ffmpeg-python
import ffmpeg

# 先頭フレーム抽出
ffmpeg.input('video.mp4').output('frame.jpg', vframes=1).run()

# OpenCVで動画を直接処理
import cv2
cap = cv2.VideoCapture('video.mp4')
ret, frame = cap.read()  # 先頭フレーム取得
```

| ライブラリ | 用途 | 特徴 |
|-----------|------|------|
| **ffmpeg-python** | ffmpegのPythonラッパー | ✅ 動画メタデータ取得<br>✅ フレーム抽出<br>✅ 多形式対応 |
| **OpenCV (cv2.VideoCapture)** | 動画のフレーム単位処理 | ✅ 高速<br>✅ 直接numpy配列で取得可能 |
| **MediaInfo (pymediainfo)** | 詳細メタデータ | ✅ コーデック、ビットレート等 |

#### **4.3.2 動画類似度計算戦略**
1. **先頭フレーム抽出** → 画像として知覚ハッシュ計算
2. **中盤フレーム抽出** → 画像として知覚ハッシュ計算
3. **終盤フレーム抽出** → 画像として知覚ハッシュ計算
4. **3つのハッシュの組み合わせで類似度判定**
5. **動画長・ファイルサイズも比較要素として使用**

## 10. パフォーマンス最適化戦略

### 10.1 スキャン処理の並列化
```python
from concurrent.futures import ProcessPoolExecutor
import multiprocessing

def scan_folder_parallel(folder_path):
    file_paths = collect_files(folder_path)
    num_workers = multiprocessing.cpu_count()
    
    with ProcessPoolExecutor(max_workers=num_workers) as executor:
        results = executor.map(calculate_hash, file_paths)
    
    return list(results)
```

### 10.2 サムネイル生成の最適化
- 初回スキャン時にサムネイルを生成してSQLiteに保存
- 2回目以降はキャッシュから取得

### 10.3 メモリ管理
- 大規模スキャン時は一括処理せず、バッチ単位で処理
- メモリ使用量: 500MB（常駐） / 1GB（ピーク）を目標
